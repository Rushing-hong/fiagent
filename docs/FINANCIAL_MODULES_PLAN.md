## 前置依赖：indicators 字段映射

**现状**：`get_financial_statements(statement="indicators")` 返回东财 `RPT_F10_FINANCE_MAINFINADATA` 原始数据，
字段名为东财内部命名，Agent 需要猜测字段含义。

**做什么**：在 indicators 返回结果中**直接附带 `normalized` 对象**——即同一个 JSON 信封里同时包含原始字段（`raw`）和标准化字段（`normalized`）。
杜邦/红旗/DCF 全部读取 `normalized` 下的标准字段名，不碰原始名。映射层与取数层耦合在同一个工具返回里，省一轮工具调用。

**映射方式**：

1. 先用 `get_financial_statements(code="600519.SH", statement="indicators")` 跑一次，拿到实际返回 JSON
2. 从真实字段名建立映射字典（key = 东财原始名, value = 标准名），同时注明每个字段的**单位**（元/万元/%、是否已×100）
3. 映射字典内置于 `get_financial_statements` 工具中，每个返回自动附带 `normalized`

**映射范围**（Day0 三票实测：600519 / 000001 / 300750，indicators 字段名一致）：

| 东财原始字段名（别名按优先级） | 标准名 | 单位 | 来源报表 |
|---|---|---|---|
| `TOTALOPERATEREVE` / `TOTAL_OPERATE_INCOME` / `OPERATE_INCOME_PK` | `revenue` | 元 | indicators / 利润表 |
| `OPERATE_COST` | `cogs` | 元 | 利润表 |
| `MLR` | `gross_profit` | 元 | indicators |
| `PARENTNETPROFIT` / `PARENT_NETPROFIT` | `net_income` | 元 | indicators / 利润表（归母） |
| `KCFJCXSYJLR` / `DEDUCT_PARENT_NETPROFIT` | `recurring_net_income` | 元 | indicators / 利润表（扣非） |
| `OPERATE_PROFIT_PK` / `OPERATE_PROFIT` | `ebit` | 元 | indicators / 利润表（营业利润近似） |
| `TOTAL_PROFIT` | `pretax_income` | 元 | 利润表（利润总额） |
| `NETCASH_OPERATE_PK` / `NETCASH_OPERATE` | `cfo` | 元 | indicators / 现金流量表 |
| `TOTAL_ASSETS_PK` / `TOTAL_ASSETS` | `total_assets` | 元 | indicators / 资产负债表 |
| `LIABILITY` / `TOTAL_LIABILITIES` | `total_liabilities` | 元 | indicators / 资产负债表 |
| `TOTAL_PARENT_EQUITY` / `TOTAL_EQUITY_PK` / `TOTAL_EQUITY` | `total_equity` | 元 | balance 优先归母；indicators 仅含少数股东口径 |
| `ACCOUNTS_RECE` / `NOTE_ACCOUNTS_RECE` | `accounts_receivable` | 元 | 资产负债表 |
| `INVENTORY` | `inventory` | 元 | 资产负债表 |
| `GOODWILL`（null→0） | `goodwill` | 元 | 资产负债表 |
| `MONETARYFUNDS` | `cash` | 元 | 资产负债表（货币资金） |
| `SHORT_LOAN+LONG_LOAN+BOND_PAYABLE+SHORT_BOND_PAYABLE+LEASE_LIAB+BORROW_FUND` | `interest_debt` | 元 | 资产负债表汇总（有息负债） |
| `EPSJB` / `BASIC_EPS` | `eps` | 元/股 | indicators |
| `ROEJQ` | `roe` | %（已×100，如 32.53） | indicators（加权） |

> 实现文件：`market/eastmoney_indicator_fields.py`。银行股 balance/cashflow 通用三表可能为空，红旗相关条目标「无法检测」或「金融股不适用」。

> ⚠️ **第1刀的前置步骤——Day 0 必须完成才能开杜邦/红旗**：
>
> 1. 跑 `get_financial_statements(code="600519.SH", statement="indicators")` → 拿到真实 JSON
> 2. 跑 `000001.SZ`（金融股）→ 验证字段一致性
> 3. 跑 `300750.SZ`（创业板）→ 验证跨板一致性
> 4. 用三只股票的真实字段名填满上表所有 `⚠️` 占位符
> 5. 将映射字典写入 `market/eastmoney_indicator_fields.py`
> 6. 修改 `get_financial_statements` 工具，在返回 JSON 中自动附带 `normalized` 对象
>
> **跨票不一致处理**：如果三只股票的同概念字段名不同（如金融股用 `FINA_*` 而一般股用另一个前缀），映射字典需支持多别名。
> **单位突变处理**：如果同一字段在某一期为 null，该期跳过不纳入计算，不使整票失败。
> **某期 null 处理**：单个报告期的某个字段为 null → 该期在用到此字段的指标上标 N/A，不影响其他期。

**ROE 自算口径**：杜邦分解使用 **期末净资产**（`net_income_t / total_equity_t`），不用平均。东财映射后的 `roe` 一般为加权平均 ROE，口径不同 → 差异 >2% 时标注"东财加权 ROE=XX%，与期末口径差异+1.5pp"，不报错。去掉了之前"差异>1%警告"的阈值——期末 vs 加权有系统性差异，不应该天天触发。

## 第1刀：杜邦分解 + 红旗自动检查

### 1A. 杜邦分解工具

**输入**：股票代码（如 `600519.SH`）。工具内部自动调用 `get_financial_statements` 取数并做 normalized 映射。**不接受 Agent 传原始 JSON**——避免 Agent 传 18 万字 JSON 浪费上下文。

**数据来源**：

- **3因子分解**：仅依赖 `indicators`（含 normalized）。需要 `net_income`、`total_equity`、`revenue`、`total_assets`。如果这4个字段在 indicators 中齐全，3因子分解立即可做。
- **5因子分解**：额外需要 `ebit` 和 `pretax_income`。如果 indicators 中有这两个字段，直接做；如果没有，自动补拉 `statement="income"`（利润表）获取。5因子标记为 `optional`——数据不够就只出3因子，不报错。

**必须输入的字段**：

| 字段 | 用于计算 | 必选 |
|---|---|---|
| `net_income` (净利润) | ROE分子 | 是 |
| `total_equity` (净资产) | ROE分母 | 是 |
| `revenue` (营业收入) | 净利率分母、周转率分子 | 是 |
| `total_assets` (总资产) | 周转率分母、权益乘数 | 是 |
| `ebit` (息税前利润) | 五因子的利息负担 | 否（缺则3因子） |
| `pretax_income` (利润总额) | 五因子的税负 | 否（缺则3因子） |

**输出**：

**3因子分解表**（每个报告期一行）：

```
ROE = 净利率 × 资产周转率 × 权益乘数

| 报告期 | ROE | 净利率(=净利/营收) | 资产周转率(=营收/总资产) | 权益乘数(=总资产/净资产) |
|--------|-----|-------------------|------------------------|------------------------|
| 2023A  |15.2%| 8.5%              | 0.85                   | 2.10                   |
| 2024A  |17.8%| 9.2%              | 0.88                   | 2.20                   |
```

同时输出每个因子的**同比变化**和**对ROE变动的贡献度**（连环替代法，固定顺序：净利率 → 资产周转率 → 权益乘数）：

```
基期 = 上一年年报（最近一期年报的上一期）
当期 = 最近一期年报
如只有一年数据（新股/新上市）→ 贡献度输出 N/A，跳过连环替代。

基期: ROE₀ = NI₀/Rev₀ × Rev₀/TA₀ × TA₀/Eq₀
替代1: ROE₁ = NI₁/Rev₁ × Rev₀/TA₀ × TA₀/Eq₀   （仅替换净利率）
替代2: ROE₂ = NI₁/Rev₁ × Rev₁/TA₁ × TA₀/Eq₀   （再替换周转率）
替代3: ROE₃ = NI₁/Rev₁ × Rev₁/TA₁ × TA₁/Eq₁   （再替换权益乘数 = 当期ROE）

净利率贡献 = ROE₁ - ROE₀
周转率贡献 = ROE₂ - ROE₁
杠杆贡献   = ROE₃ - ROE₂
合计       = ROE₃ - ROE₀ （= 当期ROE - 基期ROE，允许浮点误差±0.1%）
```

> 顺序固定为「净利率→周转→杠杆」，不可改变。验收时用同一组数据在不同运行中得到相同贡献度。
> 基期固定为"上一年年报"。只有一年数据（新股）时贡献度输出 N/A。

**5因子分解表**（可选，数据够就做）：

```
ROE = 税负 × 利息负担 × 营业利润率 × 资产周转率 × 权益乘数
    = (净利/税前利润)×(税前利润/EBIT)×(EBIT/营收)×(营收/总资产)×(总资产/净资产)
```

（第2刀 peers 完成后，可将行业ROE驱动对标——如白酒高净利率、银行高杠杆——附加到杜邦输出中。当前版本不包含此联动。）

**边界情况**：
- 净资产为负：ROE无意义，输出"N/A"，提示"资不抵债"
- 报告期内发生重大并购/资产重组：工具不自动检测（无可靠数据源）。**由 Agent 在调用前标注**——如 Agent 通过新闻/公告确认并购事件，在输入参数中设 `has_ma=true`，工具在同比列标注"可比口径可能被破坏"。Agent 不标则不做特殊处理。
- 季度数据不做杜邦分析（季节性扭曲严重），仅年度数据
- **年度过滤规则**：`REPORT_DATE` 以 `-12-31` 结尾的行才算年报。如果一年有多个年报行（如 A 股 + H 股双重披露），取最新 `REPORT_DATE`。不含一季报/半年报/三季报。
- 金融行业（银行/保险/券商）：三表结构完全不同，传统杜邦不适用。**检测规则**：用东财行业板块名匹配（含"银行"/"保险"/"证券"/"多元金融"关键词），不靠 LLM 判断。
  **金融股杜邦**：改用 ROE = ROA × 权益乘数（ROA = 净利润/总资产，期末口径）。
  **金融股红旗**：默认跳过 #2（应收暴增）、#5（存货异常）、#6（存贷双高）——这三条对银行是常态误伤，标"金融股不适用"而非"正常"。
- **只有一年数据**（新股）：杜邦分解表正常输出但无同比列，连环替代贡献度标 N/A

**金融正确性验证**：
- 3因子/5因子相乘结果应等于**自算期末 ROE**（`net_income / total_equity`），允许浮点误差±0.1%。这是恒等式校验，用来抓实现 bug，不是对东财加权 `roe` 的校验。
- 与东财映射字段 `roe` 只做差异标注（见前置「ROE 自算口径」），不因口径不同标 `validation: "failed"`。
- 若自算恒等式失败，标记 `validation: "failed"` 而非静默输出错误结果。

### 1B. 红旗自动检查工具

**输入**：股票代码（如 `600519.SH`），与 1A 对齐。工具内部自动调用 `get_financial_statements` 取 indicators（含 `normalized`）；若某条红旗所需字段缺失，**允许按需补拉** `statement="balance"` / `statement="cashflow"`（与 1A 五因子补拉 income 对齐）。补拉在工具内部完成，不把额外轮次暴露给 Agent。

**首批做这6条**：

| # | 红旗 | 检测逻辑 | 需要哪些字段 | 阈值 | 字段来源 |
|---|---|---|---|---|---|---|
| 1 | 盈利质量差 | CFO/NI < 0.5 且持续2期+ | `cfo`, `net_income` | <0.5 | indicators(净利) + cashflow(CFO) |
| 2 | 应收暴增 | 应收增速 > 营收增速 × 1.5 | `accounts_receivable`, `revenue` | >1.5x | balance(应收) + indicators(营收) |
| 3 | 扣非占比低 | 扣非/归母 < 70% | `recurring_net_income`, `net_income` | <70% | indicators(两者) |
| 4 | 商誉炸弹 | 商誉/净资产 > 30% | `goodwill`, `total_equity` | >30% | balance(商誉) + indicators(净资产) |
| 5 | 存货异常 | 存货/营收较上期年报突增>50% | `inventory`, `revenue` | >50% | balance(存货) + indicators(营收) |
| 6 | 存贷双高 | 货币/营收 > 30% AND 有息负债/营收 > 30% | `cash`, `interest_debt`, `revenue` | 两者同时>30% | balance(货币+有息负债) + indicators(营收) |

> **红旗 #1 — NI≤0 / 接近零**：净利润为负或接近零时 CFO/NI 比率失真。判定「接近零」用相对口径：`|net_income| / |revenue| < 0.1%`（`revenue` 同期；营收也为 0 则该条标「无法检测」）。此时**不看比率，改用 CFO 符号**：CFO < 0 → 触发；CFO ≥ 0 → 不触发。
>
> **异常分母（标「无法检测」，不当「正常」）**：
> - #2 / #5：上期营收或上期应收/存货为 0（或缺失）→ 增速无定义 → 无法检测
> - #3：`net_income ≤ 0` → 扣非/归母失真 → 无法检测
> - #4：`total_equity ≤ 0` → 商誉/净资产无意义 → 无法检测
> - #6：`revenue ≤ 0` → 无法检测

> 字段来源 ≠ 仅 indicators。cfo 来自 cashflow 表、应收/存货/商誉/货币/有息负债来自 balance 表、营收/净利/扣非/净资产在 indicators。缺任一字段 → 该条标"无法检测"。**补拉规则**：工具内部自动按需调 `get_financial_statements(statement="balance")` / `statement="cashflow")`，Agent 无感。
>
> **存贷双高口径注意**：使用"货币/营收"和"有息负债/营收"作为检测口径（因不敢假定能稳定拿到总资产或净资产做分母的字段一致性），此口径局限性：营收季节性强的公司可能在旺季被误触发。实务更常用货币 vs 有息负债的相对规模做判断——二期优化时改用有息负债/货币资金比。

**第二批（需要 income/cashflow 联合查询，后补）**：

| # | 红旗 | 额外依赖 |
|---|---|---|
| 7 | **（合并到红旗#1，不独立成条）** | #1 已覆盖 CFO/NI + NI≤0 的 CFO 符号判断；持续2期+的条件等价于"多期CFO为负"。二期不新增此条，只需将 #1 扩展检查窗口到更多期。 |
| 8 | 在建工程不转固 | 需要固定资产+在建工程时间序列 |
| 9 | 少数股东损益异常 | 需要利润表少数股东损益 |
| 10 | 审计意见非标 | 无法从 API 获取——需要另类数据源 |
| 11 | 研发资本化率过高 | 需要利润表研发费用+研发资本化 |
| 12 | 关联交易占比高 | 无法从 standard API 获取 |

**输出格式**：

```
### 红旗扫描结果

已检测 4 条，触发 2 条，无法检测 2 条

| # | 红旗 | 状态 | 具体数值 | 说明 |
|---|---|---|---|---|
| 1 | 盈利质量 | ⚠️触发 | CFO/NI=0.42 | 连续3期低于0.5 |
| 2 | 应收暴增 | ✓ 正常 | 应收+12%, 营收+18% | |
| 3 | 扣非占比 | ⚠️触发 | 扣非/归母=62% | 依赖非经常性损益 |
| 4 | 商誉炸弹 | ✓ 正常 | 商誉/净资产=8% | |
| 5 | 存货异常 | ✓ 正常 | 存货/营收同比年报+3% | |
| 6 | 存贷双高 | ✓ 正常 | 货币/营收=18%, 有息负债/营收=22% | |
```

**风险等级映射**：

工具只输出**触发列表 + 每条附具体数值**。不输出"买入/卖出/回避"等投资建议——那是 Agent 的解读工作。

> 工具职责边界：算出来几条触发、哪几条、每条数值多少。风险严重度判断（商誉一条 vs 造假五条权重不同）属于 Agent 的专业判断，工具不做加权综合评分。

**金融正确性验证**：
- 每条红旗的检测结果必须附带实际数值（不是 just true/false），让 Agent 和用户能做二次判断
- 阈值标注为"经验阈值，可配置"，不挂学术模型名称（这6条不是 Beneish M-Score 也不是 Piotroski F-Score，是实务常用检测项的自组合）
- 字段缺失或异常分母（见上）→ 该条标"无法检测"而非"正常"

## 第2刀：peers / 行业分位

### 2A. 板块成分获取

**输入**：A股代码（如 `600519.SH`）

**需要确定的信息**：
1. 该股票属于哪个东财行业/概念板块—— `get_sector_info(code="600519.SH", mode="membership")` 返回板块 codes 列表。
   首版直接用**东财行业板块**成分股做同业对比。**二期再做申万映射**（东财板块 ≠ 申万行业，对齐是另一个工程量）。
2. 同一板块的成分股列表（全量，不只是top N）

**数据源方案**：

`get_sector_info(code="600519.SH", mode="membership")` 返回该股票属于的板块 codes 列表（含行业和概念板块）。
**多板块时选哪块**：从 `sector_type="industry"` 的板块中，选择成分股数量在 5-200 之间且板块名称最接近股票主业的那个。优先规则：①排除名称含"综合"的行业板（太泛）→ ②取成分股数最接近中位数的板 → ③仍不明确时，支持 Agent 通过 `board_code` 参数显式指定。

然后需要能按板块 code 拉出该板块的全部成分股。push2 `get_json` 的 `fs` 参数支持板块过滤（如 `m:90+t:2` 为行业板块、`m:90+t:3` 为概念板块），按板块 code 加过滤条件。

**工程约束**：
- 全市场 push2 过滤若超时（>10秒），回退到 `screen_fundamental` top500 过滤——此时必须在输出中标注 `quality: "degraded"`，说明"板块成分取自 PE 排序 top500 子集，小盘/亏损股可能遗漏"。
- 分页：push2 默认 pageSize 足够覆盖全部 A 股板块成分（单个板块通常 < 200 只股票）。
- **如果 akshare `get_index_stocks` 可用**，作为申万对齐的备选数据源，二期考虑。

### 2B. 同业估值对比表（薄启动，首批输出）

**输入**：目标股票代码

**输出**：

```

|------|------|---------|---------|--------|---------|
| 600519 | 贵州茅台 | 22.5 | 7.8 | 28.3 | 21500 |
| 000858 | 五粮液 | 18.2 | 4.5 | 25.1 | 5800 |
| ... | ... | ... | ... | ... | ... |

行业统计:
| 指标 | 25分位 | 中位数 | 75分位 | 目标公司 | 分位排名 |
|------|--------|--------|--------|---------|---------|
| PE | 15.2 | 20.5 | 28.3 | 22.5 | 62% |
| PB | 3.2 | 5.0 | 8.1 | 7.8 | 78% |
| ROE | 12.0 | 20.0 | 28.0 | 28.3 | 82% |
```
工具只输出分位对比表。不输出"略贵/质量溢价/非泡沫"等解读——与 DCF 原则一致，解读属于 Agent。

**分位方向**（输出时在表头或脚注标明，避免 Agent 误读）：
- PE / PB：分位越高 = 相对行业越贵
- ROE：分位越高 = 相对行业盈利能力越强
- 市值：仅作可比公司表列，默认不进分位统计（避免「越大越好」误读）

**首批做这4个指标**：PE、PB、ROE、市值。这四个 push2 全覆盖（f9/f23/f37/f20），不需要跨数据源。

**营收增速**：push2 是否有对应 f 字段需要验证。如果有（如 f? 对应营收增速），加入首批；如果没有，首批砍到4个指标，二期用 indicators 的 `revenue` YoY 补充。

**二期迭代**：增加营收增速、EV/EBITDA、PEG、股息率、净利率、毛利率、资产负债率。

### 2C. 行业分位（可合并进2B或独立工具）

和 2B 是同一个东西的两个视角：
- 2B 侧重"可比公司表"（投行 comp sheet 格式）
- 2C 侧重"单公司 vs 行业分布"（一张雷达图或分位表）

合在一起做，输出时两个视角都包含。

**金融正确性注意**：
- 排除 ST/*ST 股（默认启用，避免异常值污染中位数）
- 排除 PE < 0 的亏损公司（PE 为负无意义，单独统计亏损公司数量）
- 成分股少于5只时，分位数无统计意义，只输出原始列表
- 极值处理：PE > 200x 或 < 0 的，不计入中位数计算，单独标注

## 第3刀：薄DCF

### 设计原则

**不做"一键真理"**。DCF 对假设敏感度极高（WACC 变 1%，估值变 20%+），
没有任何模型能"自动"给出正确的 WACC 和增长率。

**做"假设 → 算表"**。LLM（Agent）负责基于行业知识、公司基本面、宏观环境设定假设参数。
确定性代码负责按公式计算 FCFF 序列、折现、终值、敏感性矩阵。两者分工明确。

### 输入参数（由 Agent 设定，不是工具自动推断）

| 参数 | 来源 | 说明 |
|---|---|---|---|
| 历史营收（最近一期） | 工具从 indicators normalized 读取仅作展示 | **`revenue_t0`（基期营收）必须 Agent 显式传入**——避免 indicators 单位不一致导致 FCFF 分母错误 |
| 预测期营收增长率 | 必须 Agent 显式传入(5年数组) | 工具不提供默认值 |
| EBIT利润率 | 必须 Agent 显式传入 | 工具不提供默认值 |
| 有效税率 | 必须 Agent 显式传入 | 工具提供校验：常见 15%(高新)/25%(普通)，偏离此范围时警告 |
| 折旧摊销占营收比 | 必须 Agent 显式传入 | — |
| 资本开支占营收比 | 必须 Agent 显式传入 | — |
| 营运资本变动占营收比 | 必须 Agent 显式传入 | — |
| 无风险利率 Rf | Agent 可传，也可从 `get_yield_curve` 工具取10年期 | 默认不填，必须显式提供 |
| Beta | 必须 Agent 显式传入 | 工具提供行业参考范围提示，但不静默填入 |
| 股权风险溢价 ERP | 必须 Agent 显式传入 | A股经验 5-7%，工具不提供默认 |
| 目标资本结构 D/(D+E) | 必须 Agent 显式传入 | — |
| 债务成本 Kd | 必须 Agent 显式传入 | — |
| 永续增长率 g | 必须 Agent 显式传入 | `g ≥ WACC` → **拒绝计算**；`g > 5%` → **警告但允许继续**（与「偏离只警告、可覆盖」一致） |
| 退出倍数（EV/EBITDA） | 必须 Agent 显式传入 | 用于退出倍数法终值计算。如不传，只出永续增长法 |
| 总股本 | 从 indicators normalized 取 | 纯数据，不算假设 |
| 净债务 | 从 indicators normalized 取（有息负债-现金） | 纯数据，不算假设 |

> **红线**：增长率、利润率、WACC 所有假设参数必须由 Agent 显式传入。工具不做"行业默认值静默填入"。工具只做校验：偏离常见范围时警告，但允许 Agent 覆盖。

### 计算流程（确定性代码执行）

**步骤1：FCFF 预测（5年）**

```
对每一年 t = 1..5:
  营收_t = 营收_{t-1} × (1 + 增长率_t)
  EBIT_t = 营收_t × EBIT利润率
  NOPAT_t = EBIT_t × (1 - 税率)
  
  折旧摊销_t = 营收_t × 折旧摊销占比
  资本开支_t = 营收_t × 资本开支占比
  营运资本变动_t = (营收_t - 营收_{t-1}) × 营运资本变动占比
  
  FCFF_t = NOPAT_t + 折旧摊销_t - 资本开支_t - 营运资本变动_t
```

**步骤2：WACC**

```
Ke = Rf + Beta × ERP
WACC = Ke × (1 - D/(D+E)) + Kd × (1 - 税率) × D/(D+E)
```

**步骤3：终值 + 折现**

```
终值(永续增长法) = FCFF_5 × (1+g) / (WACC - g)
终值(退出倍数法) = EBITDA_5 × 行业EV/EBITDA倍数
                      其中 EBITDA_5 = EBIT_5 + 折旧摊销_5
                      （折旧摊销来自 FCFF 步骤1的 D&A_5）

企业价值 = Σ(FCFF_t / (1+WACC)^t) + 终值 / (1+WACC)^5
股权价值 = 企业价值 - 净债务
每股内在价值 = 股权价值 / 总股本
```

**步骤4：敏感性矩阵（固定 5×5 格点）**

```
WACC 从 (WACC_base - 1.0%) 到 (WACC_base + 1.0%)，步长 0.5%，共5列
g 从 (g_base - 0.5%) 到 (g_base + 0.5%)，步长 0.25%，共5行

           WACC →
g ↓     8.5%    9.0%    9.5%   10.0%   10.5%
2.0%    35.2    28.5    23.8    20.1    17.2
2.25%   38.1    30.8    25.5    21.3    18.1
2.5%    41.5    33.2    27.3    22.7    19.1
2.75%   45.3    36.0    29.3    24.2    20.2
3.0%    49.8    39.2    31.5    25.8    21.4
```

当前参数组合对应的单元格高亮。格点不可由 Agent 自定义——保证不同股票输出格式一致。
